# Good Case 模式提炼

本参考只记录可迁移的写作模式，不记录完整样例原文。

## 标题模式

- “机制/政策 + 影响对象”：如“重新思考加拿大对 TikTok 的监管”。
- “事件/危机 + 工具定位”：如“在格陵兰危机中定位欧盟反胁迫工具”。
- “关系重构 + 行动框架”：如“与华盛顿同行、脱离华盛顿、对抗华盛顿”。
- “政策对象 + 核心问题”：如“央行数字货币能否在无风险的情况下强化国家对公共财政的控制”。

写中文标题时优先压缩成可读的政策标题。若英文题名很长，保留主对象和主判断，删去副标题中重复的信息。

## 开头模式

可选以下进入方式：

- 背景切入：在某一地缘政治、贸易、数字治理或安全危机背景下，说明议题为何被提出。
- 判断切入：直接写“作者认为/报告指出”后的核心结论。
- 张力切入：用“并非……而是……”或“一方面……另一方面……”呈现政策矛盾。
- 结构切入：说明某一变化“不仅是工具升级，更在结构上改变……”。

开头应尽快进入原文主题，不写泛泛的“近年来国际形势复杂多变”。

## 展开模式

常见的 good case 展开方式：

- 将风险或论证归纳为二至三类，用连续句表达。
- 先写制度或政策背景，再写行为体如何利用或承受这一结构。
- 对技术、贸易、能源、防务等议题，先交代工具本身，再交代其权力关系或治理后果。
- 对国家关系议题，先说明缓和或冲突的表层变化，再说明背后的地缘政治逻辑。
- 对欧盟、德国、加拿大等政策对象，注意写出其能力约束、制度工具和可行路径。

## 收束模式

结尾通常承担政策含义，不做空泛总结。常见方式：

- “报告建议……”
- “基于此，作者主张……”
- “鉴于此，……应……”
- “因此，……需要在……之间取得平衡。”
- “这一判断意味着，……不能……而应……”

结尾要自然接上“范钶烨 摘译”。如果正文末句已经是政策建议，直接在句末加署名即可。

## 常见节奏

约 500 字摘编可按以下比例分配：

- 背景与核心判断：约 120—170 字（24-34%）
- 主要论证与关键事实：约 220—280 字（44-56%）
- 政策建议或含义：约 100—150 字（20-30%）

复杂报告可以略微增加主要论证部分，但不要为了完整覆盖所有细节而牺牲中心判断。

**字数控制总原则**：
- 先按 patterns.md 的比例分配各部分内容
- 再按 writing-style.md 的总量约束调整
- 压缩时优先删减次要案例和背景，保留核心判断和政策建议
- 扩充时优先补充论证逻辑和政策建议，不添加空泛表达
