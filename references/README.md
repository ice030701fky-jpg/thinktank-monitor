# References 目录说明

本目录包含智库摘译 skill 的参考文档，用于校准摘译质量和风格。

## 文档分工

### patterns.md
**用途**：结构模式参考

**内容**：
- 标题模式（如何构造中文标题）
- 开头模式（如何进入主题）
- 展开模式（如何组织论证）
- 收束模式（如何结尾）
- 常见节奏（字数分配比例）

**何时查阅**：
- 不确定如何组织摘译结构时
- 标题翻译不自然时
- 开头或结尾写得不够有力时

### writing-style.md
**用途**：语言风格参考

**内容**：
- 总体风格（正式、凝练、判断明确）
- 标题风格（中文化原则）
- 正文组织（段落结构、句式偏好）
- 内容取舍（保留什么、省略什么）
- 字数控制（不同要求的处理方式）
- 禁忌（常见错误）

**何时查阅**：
- 不确定用词是否合适时
- 句式不够流畅时
- 字数超标或不足时
- 不确定某个细节是否应该保留时

### quality-checklist.md
**用途**：质量检查详细指引

**内容**：
- 每个检查项的判断标准
- 常见错误示例
- 修正方法

**何时查阅**：
- 完成摘译后进行质量检查时
- 用户反馈有问题，需要定位错误类型时
- 不确定某个表达是否符合要求时

### examples.md
**用途**：完整示例和决策点标注

**内容**：
- 完整的输入输出示例
- 关键决策点说明（为什么这样处理）
- 常见错误对比

**何时查阅**：
- 第一次执行摘译任务时
- 不确定如何处理某类文章时
- 需要校准整体质量时

## 查阅顺序建议

### 首次执行任务
1. 先读 `writing-style.md`（了解总体风格）
2. 再读 `examples.md`（看完整示例）
3. 执行任务
4. 用 `quality-checklist.md` 检查

### 遇到具体问题
- 结构问题 → `patterns.md`
- 语言问题 → `writing-style.md`
- 质量问题 → `quality-checklist.md`
- 不确定如何处理 → `examples.md`

### 用户反馈后修订
1. 用 `quality-checklist.md` 定位问题类型
2. 用 `examples.md` 对照错误示例
3. 用 `patterns.md` 或 `writing-style.md` 找到正确做法
4. 针对性修正

## 维护和更新

### 何时更新这些参考

- **用户提供新的 good case**：提炼其中的可迁移模式，更新到 `patterns.md` 或 `writing-style.md`
- **用户指出新的错误类型**：补充到 `quality-checklist.md` 的常见错误部分
- **用户反馈某类文章处理不当**：在 `examples.md` 中增加该类文章的示例

### 如何更新

1. **提炼而非复制**：不要把完整的 good case 原文放入参考，只提炼可迁移的模式
2. **保持简洁**：参考文档应该是快速查阅的工具，不是教科书
3. **标注来源**：如果某个模式来自特定的 good case，可以在注释中说明（但不写入正式内容）
4. **定期清理**：如果某个模式长期未被使用或被证明无效，应该删除

### 更新示例

**错误做法**：
```markdown
## 新增 good case

原文标题：The Future of EU-China Relations
摘译：……（完整摘译内容）……
```

**正确做法**：
```markdown
## 标题模式（更新）

- "未来展望 + 双边关系"：如"欧盟与中国关系的未来"
  - 适用于前瞻性分析文章
  - 保留核心行为体，删去"The Future of"等冗余表达
```

## 注意事项

1. **这些参考来自 good case 风格提炼，不作为事实来源**：不要把参考中的案例当作可复用的事实。
2. **参考是指引，不是模板**：不要机械套用参考中的表达，要根据原文内容灵活调整。
3. **参考会过时**：如果用户的要求或偏好发生变化，应该及时更新参考文档。
4. **参考不能替代通读原文**：无论参考多么详细，都必须通读原文后再概括。
